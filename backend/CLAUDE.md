# NeuralGate — Backend CLAUDE.md
 
This file defines conventions, architecture rules, and coding standards for the NeuralGate backend.
The backend is built with FastAPI (Python), served with Uvicorn, and runs in Docker.
 
---

## What This App Does
 
NeuralGate is an enterprise internal AI gateway portal. The backend is responsible for:
- Receiving prompt submissions from the frontend `/secure` page
- Calling Cloudflare Workers AI to generate responses
- Returning structured responses to the frontend
- Health check endpoint for Render.com and Docker
The backend does NOT handle authentication — this is managed entirely by Cloudflare Zero Trust and the Cloudflare Worker at the edge.

## Folder Structure
 
```
backend/
├── main.py                         # FastAPI app entry point, mounts routers, configures CORS
├── routers/                        # Route definitions only, one file per feature group
│   ├── inference.py                # POST /api/inference
│   └── health.py                   # GET /api/health
├── services/                       # Business logic, no FastAPI imports
│   └── inference_service.py        # Calls Cloudflare Workers AI
├── schemas/                        # Pydantic request and response models
│   └── inference_schema.py         # InferenceRequest, InferenceResponse
├── core/                           # App config and settings
│   └── config.py                   # Loads env vars via pydantic-settings
├── requirements.txt
└── CLAUDE.md
```
 
---

## Folder Rules
 
- `routers/` — FastAPI route definitions only. No business logic. One file per feature group. Delegate everything to `services/`.
- `services/` — all business logic lives here. No FastAPI imports (`Request`, `Response`, `Depends` etc.). Pure async Python functions.
- `schemas/` — Pydantic models for all request bodies and response shapes. No logic, no imports from routers or services.
- `core/` — app configuration loaded from environment variables via `pydantic-settings`. Constants and shared config only.
- `main.py` — mounts all routers, configures CORS, sets up lifespan events. No business logic.

---

## Static File Serving

- FastAPI serves the React frontend from the `dist/` folder inside `backend/`
- `dist/` is generated by running `npm run build` in `frontend/` — never manually edited
- Mount in `main.py` using `StaticFiles(directory="dist", html=True)`
- The static mount must be registered LAST after all API routers, otherwise it will intercept API requests
- `dist/` is gitignored — it is always built fresh before deployment

| Type | Convention | Example |
|---|---|---|
| Files | snake_case | `inference_service.py`, `inference_schema.py` |
| Classes | PascalCase | `InferenceRequest`, `InferenceResponse` |

| Service functions | snake_case verbs | `call_workers_ai`, `format_response` |
| Config fields | snake_case | `workers_ai_api_key`, `cors_origin` |

## Router Rules
 
- Each router file creates an `APIRouter` instance and is mounted in `main.py`
- Route handlers must only: validate input (via Pydantic), call a service function, return the result
- No `httpx`, `requests`, or any HTTP calls inside router files
- Always declare explicit response models using `response_model=` on route decorators
- Use `async def` for all route handlers
```python
# Good
@router.post("/inference", response_model=InferenceResponse)
async def inference(request: InferenceRequest):
    result = await call_workers_ai(request.prompt)
    return result
 
# Bad — business logic in router
@router.post("/inference")
async def inference(request: InferenceRequest):
    async with httpx.AsyncClient() as client:
        response = await client.post(...)
    return response.json()
```

---

## Service Rules
 
- Service functions must be `async def`
- No FastAPI imports in service files (`HTTPException` is the only allowed exception)
- All external HTTP calls use `httpx.AsyncClient` with a context manager
- Raise `HTTPException` with meaningful status codes and detail messages on failure
- Services must not access environment variables directly — receive config via parameters or import from `core/config.py`
```python
# Good
async def call_workers_ai(prompt: str) -> str:
    async with httpx.AsyncClient() as client:
        response = await client.post(
            settings.workers_ai_url,
            headers={"Authorization": f"Bearer {settings.workers_ai_api_key}"},
            json={"messages": [{"role": "user", "content": prompt}]}
        )
        response.raise_for_status()
        return response.json()["result"]["response"]
```

## Schema Rules
 
- Every request body must be a Pydantic `BaseModel`
- Every response must be a Pydantic `BaseModel`
- Never return raw `dict` from route handlers
- Add field descriptions using `Field(description="...")` for documentation
- Use `model_config = ConfigDict(str_strip_whitespace=True)` on request models
```python
from pydantic import BaseModel, Field
 
class InferenceRequest(BaseModel):
    model_config = ConfigDict(str_strip_whitespace=True)
    prompt: str = Field(description="The user's prompt to send to the AI model")
 
class InferenceResponse(BaseModel):
    response: str = Field(description="The AI model's response")
```
 
---
 
## Config Rules
 
- All environment variables loaded in `core/config.py` using `pydantic-settings`
- No `os.environ.get()` calls outside of `core/config.py`
- All config fields must have types and ideally defaults or clear error messages when missing
- Import the settings singleton: `from core.config import settings`
```python
from pydantic_settings import BaseSettings
 
class Settings(BaseSettings):
    workers_ai_api_key: str
    workers_ai_account_id: str
    cors_origin: str = "http://localhost:5173"
 
    class Config:
        env_file = ".env"
 
settings = Settings()
```
 
---
 
## CORS Rules
 
- CORS configured in `main.py` only
- `allow_origins` must read from `settings.cors_origin`, never hardcoded
- In production, only allow the exact frontend origin — never use `"*"`
---
 
## Error Handling Rules
 
- Always raise `HTTPException` with a clear `status_code` and `detail` string
- Never let unhandled exceptions propagate to the client
- Use `response.raise_for_status()` on all external HTTP calls
- Wrap external calls in try/except and re-raise as `HTTPException(status_code=502)`
---
 
## Python Style Rules
 
- Python 3.11+
- Type hints on all function signatures (parameters and return types)
- No `print()` statements — use Python `logging` module
- No unused imports
- Use `httpx` for all async HTTP calls, never `requests`
---
 
## Environment Variables
 
Store in `.env` at the backend root. Never commit `.env` — commit `.env.example` instead.
 
```
WORKERS_AI_API_KEY=
WORKERS_AI_ACCOUNT_ID=
CORS_ORIGIN=http://localhost:5173
```
