# PRODUCT.md

## Users

Internal enterprise users — SRE, DevOps engineers, and security engineers who manage Cloudflare infrastructure. They use this portal during incidents, routine audits, and development work. Typical session: 10–30 minutes of focused analysis, often in dim environments (offices, on-call rooms, home offices at odd hours).

## Product Purpose

An internal AI gateway portal where authenticated users submit infrastructure prompts and receive AI-assisted analysis. Core use cases: reviewing WAF events, optimizing Cloudflare Workers, debugging edge scripts. The portal receives identity context (email, timestamp, country) injected by a Cloudflare Worker.

## Brand Tone

Professional, precise, technically confident. Not playful. Not corporate-fluff. Communicates like a senior engineer would — direct, no wasted words, functionally elegant.

## Anti-References

- SaaS dashboard clichés (hero metrics, gradient cards, icon+heading+text grids)
- Side-stripe border accents
- Gradient text for decoration
- Glassmorphism as default treatment
- Bounce or elastic easing

## Strategic Principles

1. **Dark by default** — users work in low-light environments; high contrast without glare
2. **Motion serves clarity** — animations guide attention, never distract
3. **Minimalist but not empty** — every element earns its place
4. **Brand orange (#FF6633) as accent only** — not overwhelming, used for CTAs and key indicators

## Register

product — this is an app UI / internal tool where design serves the product
